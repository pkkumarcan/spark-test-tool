## 1) Quickstart (Golden Path)

### Goal
Validate that all generated image assets are fully formed, match requested dimensions, and are free of rendering artifacts before moving to video generation.

### When to run this chapter
- Instantly after generating images from visual scripts (Chapter X42 / X44).
- Before running video synthesis or stitching phases.

### Default steps (golden path)
1) Scan all image assets inside `/jobs/<job_id>/03_images/`.
2) Load each image using Pillow to check resolution and color mode (RGB/RGBA).
3) Perform visual check using multi-modal LLM to verify there are no deformed elements or broken outputs.
4) Write findings to `image_qc_report.json`.

### Done looks like
- [ ] Output exists: `/jobs/<job_id>/99_logs/X47/image_qc_report.json`
- [ ] QC passed: All image assets flagged as `valid` in the report.
- [ ] Manifest updated: `job_manifest.json -> pipeline_steps[X47]`

## 5) Procedure (Operator Steps)

### Step 1 — Basic Format Inspection
- **Inputs:** Generated image path.
- **Action:**
  - Execute a Python checker script that loads the file using PIL:
    ```python
    from PIL import Image
    try:
        with Image.open(image_path) as img:
            img.verify()
    except Exception as e:
        print(f"Corrupted: {e}")
    ```
- **Expected output:** Returns no exception (successful validation).
- **Common failures:** Truncated file or zero-byte write.
- **Fix:** Regenerate the specific scene index with a different seed.

## 7) Smoke Tests (Mandatory)

### Smoke Test A — Minimal (fast)
- **Goal:** Run format check on a known valid and invalid image file.
- **Input:** Test directory with 1 sample image.
- **Steps:** Load and read dimensions via Pillow.
- **Pass criteria:** Dimensions returned match expected width/height.
- **If fails:** Check if file is readable or if PIL installation is broken.

## 8) QC Checklist + Scoring

### QC checklist (tick-box)
- [ ] No file corruption or file read errors.
- [ ] Resolution matches config specifications exactly.
- [ ] File size is > 50KB (failsafe against tiny black images).

### Scoring rubric (1–10)
- **Quality:** 10 if all images are perfectly formed and clean; 1 if any image is corrupt.
- **Stability:** 10 if parser executes reliably without locking files.

## 10) Troubleshooting (Top Issues)

### Issue 1 — Zero-byte Image Files
- **Cause:** ComfyUI container was killed or lost connection during final output write.
- **Fix:** Verify ComfyUI URL is responsive, delete zero-byte files, and rerun generation step.
